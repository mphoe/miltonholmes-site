Developer Tools Guide

Pre-requisites

Make sure you're using Google Chrome as your web browser.

This guide will assume you are using a Windows PC (some keyboard commands are different on Mac, and Mobile doesn't have them at all)

Opening the Developer Tools

You can open the developer tools by either pressing F12 or Ctrl, Shift,i (which will open on the Elements tab) Alternatively, right-click and choose "Inspect".

To reposition the developer tools, click the three-dot button in the top right corner and choose one of the symbols next to Dock Side (pictured). I'm a fan of the first and third options - seperate-window and bottom-docked.

Elements Tab

The Elements tab lets you look at all of the HTML elements that make up the webpage you're on. On the right you can see a window with Styles (CSS) in it, you can also edit this.
To select Elements, right click on them and select "Inspect" - or press Ctrl, Shift,C and left click on the element you wish to inspect.

Note: Any changes you make in the Elements tab are only affecting the page that your web browser has downloaded. When you refresh (by pressing F5 maybe) your browser bins your local version and downloads the page again.

Tasks

  1. Inspect the box that says "Inpect Me!" - the element should appear in the Elements tab
  2. Double click on the inner HTML (the text "Inspect Me!") and change it to something else. - notice that the box has changed on the webpage
  3. Double click on the Class (the text "alert alert-warning") and replace "alert-warning" with "alert-success"

Extension tasks

  • Edit the CSS rules in the class of "alert-success" and change the background-color - you can do this in the Styles window.
  • Find the secret span hidden in one of the boxes, and unhide it - HINT: it has the class "hidden"

Console Tab

The console is really cool, it allows you to see logs, errors and other messages - and change the webpage with javascript in real time. The first thing you'll notice is a console log saying "Hello World" - I made this in the devtools.js file that is imported into this page. We'll come to this later.

You can type things into the console by clicking on the line that starts with > - then just type anything javascript.

Note: You can toggle a Console window beneath any Dev Tools tab by pressing Esc - sometimes it's helpful to have the console available wherever you are.

Tasks

  1. Type console.log("hello there"); into the console. - notice that the message is printed out.
  2. Type console.error("help me"); into the console. - the message is printed out and coloured red, because it's an error!
  3. Type document.getElementById("anchor").innerHTML = "I'm the inner HTML"; into the console. - Can you see it appear on the page?
  4. Type document.getElementById("anchor").className = "alert alert-success"; into the console. - Notice how the text has now become a green box!

If you've done the last two tasks, you've just used Javascript to get the element with an id of "anchor" and then change the text inside it and it's CSS classes - which is pretty cool! document.getElementById is something I use all the time to get bits of the page and then modify them.

Note: Just like the Elements Tab, any changes you make in the Sources tab are only affecting your local page, and aren't permanent.

Sources Tab

This tab lets you look at files used by the current page. The left-hand window shows you all the folders and files (HTML, CSS, Javascript) the page is using. The middle window displays the file you currently have selected. The right-hand window contains a lot of complicated things like the Call Stack that I won't go into here.

Task

Look inside "scripts" for devtools.js - it's the javascript file that this webpage uses. See if you can find where the "Hello world" console log is. I've stuck in a bunch of comments (beginning with // and in green) to help you understand.

Network Tab

This tab will monitor all network activity that happens on the webpage. This includes:

  • HTML, CSS, Image and Javascript files that make up the page
  • Requests to and from the server (not currently featured in this page)
  • Streaming of videos - watch a video on youtube and see the chunks downloaded bit by bit

Once you've started recording (this should happen when you open the Network tab) and refreshed the page (by pressing F5 maybe) the tab will have a timeline running along the top and a table showing everything that's happening on the network (as mentioned above). I'll explain what each column means:

Name
The file/request's name
Status
The HTTP status code. 200 codes mean success, 400 codes (such as the legendary 404 "not found") are errors. Find out more here: Wiki Page for HTTP codes
Type
The type of file being requested - could be for a javascript file or xhr (a GET request for information) for example
Initiator
What made the request? Was it the web page (so "devtools" in this case) or was it one of the scripts?
Size
How big the request/response was in bytes
Time
How long the request took overall
Waterfall
A visual breakdown of how long the request took. Hover over a blob to see exactly how long each bit of the request (such as Waiting (TTFB) - for the server to respond)

Application Tab (Specifically Cookies)

This tab is pretty complicated, and I'm only going to introduce you to one part of it - Cookies. In the window on the left-hand side of the tab, look under Storage for Cookie. If you expand that, you'll see the current website and any cookies you're storing for it.

What are cookies?

Cookies are small pieces of information that the webpage stores locally on your web browser, so that when you come back to the site it can read them and "remember" things - such as what items you had in your cart on a shopping website. Cookies have a name, a value and an expiry time (so you can tell them how long to stick around for)

For Example: At the top of this page is a "Pre-requisites" card. If you click the "Got it" button it will hide the card and create a cookie called "cardClosed" that tells the page you've closed the card. The page checks for this cookie every time you load up, so the card won't appear if you refresh the page. Don't worry though! the cookie expires after 10 minutes, so the card isn't gone forever.

You can look at and delete the cookie in the Application tab. (As seen in the picture below)

The other tabs

You might be wondering why I haven't covered the Performance, Memory, Security and Audits tab. This is because I don't know much about them. I'll sum up what I know below:

Performance tab
This will record your use of the webpage and tell you how much time was spent Loading it, running Scripts and Rendering the page.
Memory tab
You can take snapshots of how much memory (RAM) the webpage is currently taking up - breaking it down into a whole bunch of things that I don't understand
Security tab
Here you can check out how secure your connection to the website is. This includes what protocol you're using (TSL 1.2 in this case) and what certificate you have. If you View Certificate, you'll see i've got a certificate for this site, which tells your browser that it's trustworthy.
Audits tab
This will look through your page and tell you how you could make it better. My page for example isn't making great use of the CSS rules it can access. (this is because i'm using Bootstrap, which has a huge number of rules, an dthe idea is that you pick and choose)
